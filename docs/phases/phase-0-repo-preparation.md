# Phase 0 — Repository Preparation

## Goal

Create a clean, implementation-ready baseline for BlueNote.

## Delivered baseline

- Git hygiene and ignore rules
- runtime/tooling config for Bun-first development
- docs split into product/architecture/workflow/phase references
- initial managed-root directory layout
- CI baseline and smoke checks

## Current status

This is a historical foundation phase. The current public repository now includes the CLI, storage, search, migration, TUI, and smoke/test coverage described by later phase docs and the README.

## Non-goals

- no AI or sync features
- no hosted backend or cloud-only assumptions
